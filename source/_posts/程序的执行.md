在此之前，我们学习了程序的生命周期、程序的实质（指令序列）、计算机的硬件构造。现在是时候我们开始学习程序具体是如何执行的。

在之前讲述 ISA 和运行 hello 程序将 “hello world!” 打印到显示屏时，我们可以看到：

- 不同指令集架构，指令格式不同。程序执行首先要对指令格式“解析”。
- CPU 是指令执行的核心。程序的执行和 CPU 的构造密切相关。
- 程序相关数据在存储器、主存、寄存器中传递。因此，数据的路径（数据通路）也是程序执行的关键之一。

因此，接下来我们会就程序/指令执行、CPU 构造、数据通路进行介绍。

## 一、程序/指令的执行

计算机功能通过程序的执行实现，程序是由指令组成。CPU 通过对执行一条条指令完成功能的实现。所以明白指令是如何执行的，就明白了程序是如何运行的。下图是指令执行的流程，来自袁春风书籍《计算机系统基础》P223.

![图片来自袁春风书籍《计算机系统基础》](https://note-1310215887.cos.ap-nanjing.myqcloud.com/img/image-20220328135802108.png)

一条指令的执行过程总结起来，有五个过程：取指、译码、执行、访存、写回。但详细起来，可以有：取指令、指令译码、计算源操作数地址并取操作数、执行数据操作、计算目的操作数地址并存结果、计算下条指令地址等步骤。

- **取指令（fetch）**

  马上要执行的指令的地址存放在程序计数器（PC）内。取指令操作是将 PC 指向的存储单元中的指令加载到指令寄存器中（IR）。

- **译码（decode）**

  译码阶段包括：指令译码、计算源操作数地址并取操作数、计算目的操作数地址。

  **指令译码：**不同指令功能不同，指令的执行过程也会不同。所以需要按照指令集规定的格式对指令进行译码，也就是将指令“翻译”成正确的含义。在这个过程中，可以识别出这条指令的操作类型、操作数以及寻址方式。

  **计算源操作数地址：**根据寻址方式确定源操作数地址计算方式，若是存储器内数据，则需要一次或多次访存；如果是寄存器的数据，可以直接进行下一步数据操作步骤。

  **计算目的操作数地址：**在该阶段，和计算源操作数地址一样，也可以确定目的操作数地址。

- **执行（execute）**

  对指令进行真正运算的过程。此阶段的任务是完成指令所规定的各种操作，具体实现指令的功能。如加法操作、乘法操作等。

- **访存（memory access）**

  根据指令需要，有可能要访问主存。或将数据从存储器中读出，或者写入存储器。

- **写回（write back）**

  指将指令执行的结果写回通用寄存器组的过程。如果是普通运算指令，该结果值来自于“执行”阶段计算的结果：如果是存储器读指令，该结果来自于“访存”阶段从存储器中读取出来的数据。

## 二、CPU 构造

来自袁春风书籍《计算机系统基础》

![在这里插入图片描述](https://note-1310215887.cos.ap-nanjing.myqcloud.com/img/20191020215216922.png)

其实，上述的很多部件上次都讲过，三条总线可能会陌生一些。**地址总线、数据总线、控制总线**是用来 CPU 和 外设进行传递信息的，信息主要包括数据（包括数据和指令）、地址、控制信号三类。

下面按照上图来仔细讲解 CPU 部件：

- PC 寄存器：存放即将执行指令的地址。PC 中的地址一般是 PC + “1” 得到的。但如果指令是转移类指令（如 j 指令）就会经过 j 指令规定的方式计算下条指令的地址并存放在 PC。
- 指令寄存器（IR）：存放即将执行的指令。
- 译码寄存器（ID）：对指令寄存器中的操作数进行“翻译”。
- 总线控制逻辑：实现对总线的控制。控制数据和地址的传输等。
- 中断机制：实现异常、中断的处理。

## 三、数据通路

指令执行过程中数据经过的路径称为**数据通路**。从之前学过的可以知道，数据通常会经过 ALU、通用寄存器、状态寄存器、Cache被称为**功能部件**，也是数据通路的一部分。

### 3.1 单总线数据通路

![数据通路与总线结构 硬布线控制器_计算机组成原理_10](https://note-1310215887.cos.ap-nanjing.myqcloud.com/img/c913cf64eafb076561f7cb1ae123c32b.png)

图片来自[数据通路与总线结构 硬布线控制器](https://blog.51cto.com/u_15298563/3087593)

从图中就可以看出，单总线数据通路就是用一组总线连接整个计算机各大功能部件，功能部件之间的信息传输都会通过这组总线。使用这种方式，优点是设计简单；缺点是会导致竞争资源。

在单总线数据通路结构中，有四种组成所有指令功能的基本操作（考虑到初学，下面代码没有关于控制信号的操作）：

1. 寄存器与寄存器间传输数据

   例如将数据从 PC -> DR：

   ```
   PC -> Bus
   Bus -> DR
   ```

2. 算数/逻辑运算

   从内存和寄存器各取一个值，相加结果写入另一个寄存器。

   ```
   Ad(IR) -> Bus -> AR，   
   MEM -> Bus -> DR    
   DR -> Bus -> X -> A    
   R0 -> Bus -> B   
   A + B -> Z -> Bus -> R1      
   ```

   

3. 主存和 CPU 之间的数据传送

   CPU 从 men 取指令操作：

   ```
   PC -> Bus -> AR
   MEM(AR) -> DR
   DR -> Bus -> IR
   ```
