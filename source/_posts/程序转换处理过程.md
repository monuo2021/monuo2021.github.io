一个由高级语言编写的程序，必须转换成二进制代码，才能在机器上运行。这种可以在机器上运行是二进制代码被称为**可执行目标文件**。

![源程序处理过程](https://gitee.com/tanliang-TL/blogpic/raw/master/img/sp20210723_083211_152.png)

上述过程可分为四步：

- **预处理**
- **编译**
- **汇编**
- **链接**

## 一、预处理

预处理过程主要**处理源文件中以“#”开头的预编译指令**，如：

- 删除“#define”并展开所定义的宏。
- 处理所有条件预编译指令，如“#if”,“#ifdef”, “#endif”等。
- 插入头文件到“#include”处，可以递归方式进行处理。
- 删除所有的注释“//”和“/* */”。
-  添加行号和文件名标识，以便编译时编译器产生调试用的行号信息。
- 保留所有#pragma编译指令（编译器需要用）

经过预处理后得到的是 **.i 文件**（预处理文件），该文件是一个可读的文本文件 ，但不包含任何宏定义。

### 预处理命令 

```shell
gcc –E hello.c –o hello.i
```

或者

```shell
cpp hello.c > hello.i
```

## 二、编译

编译过程就是将预处理后得到的预处理文件（如 hello.i）进行一系列分析，**生成汇编代码文件**的过程。用来进行编译处理的程序称为**编译程序（编译器，Compiler）**

编译包括以下过程：

- 词法分析
- 语法分析
- 语义分析
- 中间代码生成
- 代码优化

经过编译后，得到**.s文件**（汇编代码文件）还是可读的文本文件，CPU无法理解和执行它。

### 编译命令

```shell
gcc -S hello.i -o hello.s
```

或

```shell
gcc -S hello.c -o hello.s
```

或

```shell
/user/lib/gcc/i486-linux-gnu/4.1/cc1 hello.c
```

## 三、汇编

汇编是**将汇编语言源程序转换为机器指令序列**，每条汇编语句几乎都对应一条机器指令。汇编过程相比编译较为简单，没有复杂的语法，没有语义，也不需要指令优化，只需根据汇编指令和机器指令的对照表一一翻译即可。

汇编结果是一个 **.o 文件（可重定位目标文件）**，其中包含的是不可读的二进制代码。

### 汇编命令

```shell
gcc –c hello.s –o hello.o
```

或

```
gcc –c hello.c –o hello.o
```

或（as是一个汇编程序）

```
as hello.s –o hello.o
```

## 四、链接

链接过程**将多个可重定位目标文件合并以生成可执行目标文件**。

![sp20210723_101021_361](https://gitee.com/tanliang-TL/blogpic/raw/master/img/sp20210723_101021_361.png)

### 链接命令

```shell
gcc –static –o myproc main.o swap.o
```

或

```
ld –static –o myproc main.o swap.o
```

其中，static 表示静态链接，如果不指定 -o 选项，则可执行文件名 为“a.out”。

**a.out 就是最终的可执行文件。**
